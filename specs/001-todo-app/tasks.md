**작성일**: 2026-07-15  
**기반 문서**: [spec.md](spec.md), [plan.md](plan.md)

## Phase 1: 준비 및 기본 인프라

**목적**: 앱이 동작할 기본 구조와 저장소 연결을 준비한다.

- [ ] T001 프로젝트 구조를 생성하고 app 패키지, templates, tests 디렉터리를 구성한다.
  - 완료 조건: app/main.py, app/models.py, app/schemas.py, app/crud.py, app/database.py, app/templates/index.html, tests/test_api.py 파일이 생성되어 있다.
- [ ] T002 SQLite 연결과 세션 관리 기반을 구현한다.
  - 완료 조건: app/database.py에서 DATABASE_URL 환경변수를 읽고 SQLite 세션을 생성할 수 있다.
- [ ] T003 [P] Todo ORM 모델과 Pydantic 스키마를 정의한다.
  - 완료 조건: Todo 모델과 생성/응답 스키마가 정의되었고, 제목/완료 상태/생성 시각 필드가 포함되어 있다.

---

## Phase 2: 기본 API 구현 (P1)

**목적**: 할 일 생성·조회·완료 토글·삭제 API를 독립적으로 구현한다.

- [ ] T004 목록 조회 API를 구현한다.
  - 완료 조건: GET /api/todos가 정상 동작하고, status=all|active|completed에 따라 필터링된 목록을 반환한다.
- [ ] T005 생성 API를 구현한다.
  - 완료 조건: POST /api/todos가 올바른 제목으로 새 Todo를 생성하고, 빈 제목/공백/100자 초과 시 422 응답을 반환한다.
- [ ] T006 완료 토글 API를 구현한다.
  - 완료 조건: PATCH /api/todos/{id}로 completed 상태를 변경할 수 있고, 존재하지 않는 id는 404 응답을 반환한다.
- [ ] T007 삭제 API를 구현한다.
  - 완료 조건: DELETE /api/todos/{id}가 정상 삭제를 수행하고, 존재하지 않는 id는 404 응답을 반환한다.
- [ ] T008 [P] 상태 확인 API를 구현한다.
  - 완료 조건: GET /health가 {"status":"ok"}를 반환한다.

## Phase 3: 남은 개수 및 필터 로직 (P1)

**목적**: 남은 개수와 필터 기준이 정확히 동작하도록 구현한다.

- [ ] T009 미완료 항목 수 집계 로직을 구현한다.
  - 완료 조건: 미완료 항목 수가 정확히 계산되어 API 응답/화면 렌더링에 반영된다.
- [ ] T010 [P] 필터 로직을 구현한다.
  - 완료 조건: status=all, active, completed에 따라 목록이 올바르게 필터링된다.

## Phase 4: 화면(UI) 구현 (P2)

**목적**: API 완성 이후 사용자 화면을 구현한다.

- [ ] T011 메인 화면 템플릿과 기본 레이아웃을 구현한다.
  - 완료 조건: GET /가 HTML 화면을 반환하고, 입력창/목록/필터/남은 개수 영역이 렌더링된다.
- [ ] T012 [P] 화면에서 fetch 기반 생성·완료 토글·삭제 동작을 연결한다.
  - 완료 조건: 사용자가 입력·완료 토글·삭제 동작을 수행하면 화면이 즉시 갱신되고, 남은 개수와 필터 결과가 반영된다.
- [ ] T013 빈 상태와 반응형 스타일을 구현한다.
  - 완료 조건: 할 일이 없을 때 안내 문구가 표시되고, 360px 폭 화면에서도 주요 UI가 정상적으로 보인다.
